CMS Blue Button® API

The Centers for Medicare and Medicaid Services (CMS) Blue Button API is a developer-friendly, standards-based, data API that enables Medicare beneficiaries to connect their Medicare claims data to the applications, services, and research programs they trust.

Join the Developer Preview

CMS is inviting developers to provide feedback on the new CMS Blue Button API for the purpose of improved development. Participants in the Developer Preview can build integrations with their software using sample claims data.

  • Enables a beneficiary to grant your app access to their data via OAuth2
  • Gives your app access to three years of Part A, B, and D claims data for a beneficiary
  • Uses HL7 FHIR specification

Using the API

During the Developer Preview, you will be able to query a sandbox environment populated with fictitious beneficiary data.

Base Request URL:

https://dev.bluebutton.cms.fhirservice.net

Authorization:

The authorization and token URL endpoints implement vanilla OAuth2.

HTTP  GET /o/authorize  
HTTP  POST /o/token

API Endpoints:

  • Get Profile for an Individual Beneficiary
  • Get Explanation of Benefit for an Individual for Beneficiary
  • Get Patient Record for an Individual Beneficiary
  • Get Coverage for an Individual Beneficiary

In each of these API Endpoints, the "sub" value is unused since the client can only return data owned by the access token (i.e. her or his own data).

If the _format option is omitted, then JSON is returned by default.

As a security measure the date of birth, SSN, and HICN are not provided.

Get Profile for an Individual Beneficiary

HTTP  GET /connect/userinfo

The above URL fetches the beneficiary's basic account information. This is most often used when creating an account within your application. An HTTP GET is called and the response is returned as JSON.

        
        {
          "sub": "fflinstone",
          "prefered_username": "fflinstone",
          "given_name": "Fred",
          "family_name:, "Flinstone,
          "name": "Fred Flinstone",
          "email": "pebbles-daddy@example.com",
          "created": "2017-11-28",
          "fhir_id": "123456789",
        }
